# 截图与视觉证据审阅

## 1. 为什么必须实际查看截图

实验报告中的截图不是装饰，而是执行证据。图注和正文可能描述“配置成功”，但截图里可能仍有报错、参数不一致、时间不匹配或关键区域被裁掉。OCR 只能帮助定位文字，不能代替视觉判断，也不能可靠理解窗口状态、拓扑关系、颜色图例和布局。

每张重要图片都要区分三种信息：直接可见的内容、结合上下文得到的推断、由于分辨率或裁剪而无法确认的内容。回复中不得把后两类写成确定事实。

## 2. 通用检查框架

首先判断图片试图证明哪一项任务。然后检查关键对象是否完整出现，文字与数值是否可读，界面状态是否清楚，截图是否包含必要上下文，图注是否准确，正文是否引用并解释了图片。最后检查隐私与安全信息。

一张有效截图通常同时回答“在哪里操作、操作了什么、结果是什么”。如果只能回答其中一个问题，可能需要配套文字或另一张截图。截图数量不是质量指标；重复的全屏图片会降低报告可读性。

## 3. 终端、命令行与抓包截图

检查命令本身、关键参数、目标对象、输出或退出状态、错误信息和提示符上下文。对网络实验，还应核对源与目的地址、接口、协议、端口、时间和过滤条件。只显示最后一行“success”通常不足以证明前面的配置正确。

抓包截图要能看清过滤表达式、数据包列表中的关键字段以及必要的详细字段。若正文讨论握手、时序或重传，应有多个相关数据包或序列号证据，而不是单个包的局部截图。

## 4. 代码、数据库与开发工具截图

代码截图要能定位文件或函数，并包含与结论有关的上下文。运行结果应和代码、输入或测试用例对应。数据库截图要核对当前库、表、SQL 语句、结果列与行数；只展示结果表而不展示查询，可能无法证明如何得到结果。

编译器或 IDE 截图要区分“程序成功运行”和“只完成编译”。错误或警告不能被裁掉。若报告声称通过测试，应展示测试命令、用例数量和结果摘要，必要时附失败用例的修复说明。

## 5. 图表、仪器与仿真截图

图表应有坐标轴、单位、图例、数据范围和必要标题。正文中的趋势或比较必须和图中可见变化一致。仪器或仿真软件截图应能看清量程、通道、参数、连接或模型状态；数值没有单位时不能可靠解释。

如果截图中的结果明显超出合理范围，先标记异常并建议检查量程、单位、输入、边界条件和数据处理，不要直接替用户决定应该改成哪个“正确数值”。

## 6. 清晰度和裁剪

建议截图使用原始分辨率，避免聊天软件二次压缩。正文中显示过小的图片可以保留原图附件或适当裁剪，但裁剪必须保留窗口标题、关键对象和结果上下文。多个重点区域可拆成编号子图，并在图注中解释各自用途。

模糊图片的反馈要具体：指出无法辨认的是命令、参数、数据、坐标、错误码还是状态栏，并说明重新截图时应保留什么区域。不要笼统地说“图片不清楚”。

## 7. 图文匹配

核对图号、图注、正文引用和截图内容是否一致。常见错误包括图号顺序混乱、正文说“如图 3”但图 3 是另一任务、截图中参数与正文不一致、图注只写“运行结果”而没有说明对象和条件。

好的图注可以写成“图 4 配置静态路由后从主机 A 到主机 B 的连通性验证，丢包率为 0%”。如果结果无法直接从图中看到，图注不能替代证据。

## 8. 隐私与安全

检查密码、API Token、Cookie、私钥、访问链接、身份证号、手机号、个人邮箱、真实内网凭据和不必要的内部地址。发现后建议使用遮盖后的截图替换。反馈中不要再次完整抄录这些值；只说明所在图片与需要遮盖的字段。

姓名、学号等课程要求字段可以保留，但公开分享前仍应考虑匿名化。不要因敏感信息存在而忽略实验内容审阅，应把它列为提交前的高优先级修复项。

## 9. 截图反馈格式

对每张关键图使用简洁记录：

```text
图/页：图 4，连通性验证
直接可见：命令目标为 192.0.2.10，四次响应，丢包率区域可读
可证明：该时刻主机到目标地址可达
不能证明：此前路由配置完全正确，因为配置命令和路由表未展示
修改建议：补充路由表或关键配置截图，并在正文解释验证条件
```

对于足够清楚且能支撑正文的截图，可以写“证据充分，无需修改”，不必强行要求更多图片。

## 10. Local Contact Sheet and Context Map

`inspect_report.py` records nearby DOCX paragraph text for each embedded image and creates `contact-sheet.jpg` when Pillow is available. The contact sheet is only an overview: the agent must still open important source images individually. Context text helps map a screenshot to a section but never substitutes for visual evidence.
